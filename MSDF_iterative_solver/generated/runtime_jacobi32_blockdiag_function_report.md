# Runtime Jacobi32 Block-Diagonal Single-Iteration Functional Report

This report records the first 32-active-row single-iteration functional
checkpoint for the runtime-loadable iterative-solver top.

## Scope

The tested workload is a cluster-local Jacobi-family fixture:

| item | value |
| --- | --- |
| Matrix shape | `32 x 32` |
| Physical mapping | `8` clusters x `4` rows/cluster |
| Active rows | `32` |
| Degree per row | `4` fixed slots |
| Coefficient format | `Q2.6`-style signed rail template, `BIT_WIDTH=8` |
| Runtime top | `iter_dense_small_runtime_top` |
| Runtime storage | `runtime_mem_style=1`, RAM-wrapper field banks |
| Certification | per-cluster `block_H`, `BLOCK_SIZE=2`, `ACC_WIDTH=24` |
| Initial state used for arithmetic | zero state |

The fixture is generated by:

```bash
conda run -n qas python MSDF_iterative_solver/make_jacobi32_blockdiag_runtime_vectors.py
```

Generated vectors are stored under:

```text
MSDF_iterative_solver/generated/rtl_vectors/jacobi32_blockdiag/
```

## Why Block-Diagonal

The current runtime top supports cluster-local source row indices. A generic
banded `32 x 32` Jacobi matrix has cross-cluster dependencies and therefore
needs a future inter-cluster source router or halo-window mechanism. This
checkpoint intentionally uses `8` independent `4 x 4` Jacobi blocks, so it
exercises the routed `NUM_CLUSTERS=8` / `32` active-row hardware without
violating the current RTL contract.

This is a real runtime-loader/workload test, not a proof that arbitrary
cross-cluster sparse matrices are already supported.

## Checked Path

The testbench is:

```text
MSDF_iterative_solver/tb/tb_iter_dense_runtime_jacobi32_blockdiag.v
```

It checks the following path:

1. runtime writes for all `8` template payloads;
2. runtime writes for all `8` certification payloads;
3. active window load from field-bank RAM wrappers;
4. ping-pong state-load port write/readback;
5. row scheduler active mask for all `32` rows;
6. one full iteration through row-update, delta, block-bound, `block_H` certification, and iteration controller;
7. per-cluster `max_error` and `certified` flags against generated golden vectors;
8. loader/window/iteration counters.

The state-load path is first checked with a non-zero row-0 write per cluster.
Before arithmetic launch, those rows are overwritten to zero so the iteration
matches the generated first-iteration Jacobi golden vectors.

## Current Limitation

The current top still lacks inter-cluster source routing. To support a generic
Jacobi/PageRank sparse tile, the next architectural step is one of:

1. halo-window source replay between adjacent clusters;
2. a cluster source crossbar with bounded degree;
3. a static sparse schedule that groups rows so each active window remains
   cluster-local.

Until one of those is implemented, `jacobi32_blockdiag` is the valid large
functional checkpoint, while the existing raw `jacobi32_matrix.json` remains a
future cross-cluster routing test.
