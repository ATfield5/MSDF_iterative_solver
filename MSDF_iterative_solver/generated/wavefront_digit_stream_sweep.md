# Wavefront Digit-Stream Sweep

This report is generated by `run_wavefront_digit_stream_sweep.py`.
It uses the same directed radius-1 multistage cluster testbench and
compares a live wavefront pipeline against a sequential full-wait
reference that restarts one complete stage at a time.

Interpretation:

- Speedup grows with the number of streamed solver stages because the
  full-wait reference pays the row-engine fill/drain cost at every
  stage boundary.
- The wavefront path still runs all digits; it is not an early-stop or
  reduced-precision result.
- This is an Icarus functional/cycle checkpoint only.  Runtime mode3
  integration and U55C route are still required before claiming a
  routed solver-level throughput result.
