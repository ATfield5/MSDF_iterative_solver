# Runtime Jacobi32 Global-Source Solver Report

This report records the first inter-cluster source-routing checkpoint for the
runtime-loadable iterative-solver top.

## Scope

| item | value |
| --- | --- |
| Matrix shape | raw banded `32 x 32` Jacobi fixture |
| Physical mapping | `8` clusters x `4` rows/cluster |
| Active rows | `32` |
| Source indexing | active-window global row index |
| Source index width | `5` bits |
| Replay mode | `global_source_replay=1` |
| Iterations checked | `6` |
| Runtime storage | `runtime_mem_style=1`, RAM-wrapper field banks |

The fixture is generated by:

```bash
conda run -n qas python MSDF_iterative_solver/make_jacobi32_global_runtime_vectors.py
```

Generated vectors are stored under:

```text
MSDF_iterative_solver/generated/rtl_vectors/jacobi32_global/
```

## What Changed

The previous block-diagonal checkpoints were cluster-local: each row could only
read source rows inside its own `4`-row cluster. This checkpoint adds a
compatibility-preserving global-source path:

```text
template payload source field: local 2-bit index -> global 5-bit index
state replay source rows: one cluster -> all 32 active rows
```

The default RTL path remains cluster-local. Global-source replay is enabled only
when `src_idx_width=5` and `global_source_replay=1`.

## U55C Routed Checkpoint

Command shape:

```bash
MSDF_NUM_TOTAL_CLUSTERS=8 \
MSDF_NUM_CLUSTERS=8 \
MSDF_SRC_IDX_WIDTH=5 \
MSDF_GLOBAL_SOURCE_REPLAY=1 \
MSDF_RUNTIME_MEM_STYLE=1 \
MSDF_CLK_PERIOD_NS=5.000 \
MSDF_OOC=1 \
MSDF_RUN_ROUTE=1 \
vivado -mode batch -source MSDF_iterative_solver/synth_iter_dense_small_runtime_top.tcl
```

Routed result:

| path | WNS | LUT | FF | CARRY8 | DSP | BRAM | Dynamic W | Total W |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| cluster-local NC8 | `+0.643 ns` | `10267` | `7520` | `396` | `64` | `8.5` | `0.620` | `3.905` |
| global-source NC8 | `+0.028 ns` | `35495` | `8495` | `652` | `64` | `9.5` | `1.903` | `5.216` |

The global-source path passes timing, but it is routing-dominated. The worst
path starts at one cluster's ping-pong bank select and ends inside another
cluster's row-update input register. Reported data-path delay is `4.954 ns`,
with `75.797%` from routing. This is expected for a full active-window source
mux.

## Interpretation

This checkpoint proves that raw banded `32 x 32` Jacobi can be represented and
executed by the runtime top with inter-cluster source replay. It should not be
used as the final paper architecture because the full global mux costs too much
LUT and routing slack.

The next implementation step should replace global-source replay with a
halo-window replay fabric:

```text
cluster-local rows + bounded neighbor halo rows
```

That preserves the ability to run banded matrices while avoiding an all-to-all
source mux across the full active window.
