## Current Paper-Candidate Conclusions

- P1 native serial-serial 32-input online scaling is not practical on U55C in
  the current fully parallel wavefront shape: synthesis estimates exceed the
  device LUT budget.
- P2/P3 demonstrate that parallel-in bounded affine rows can use `online_delay=2`
  for the PageRank-style fixture while preserving a 32-digit external output.
- P2 degree4/physical8 is U55C `5 ns` route-clean.  P2 degree8/physical8 now
  has a timing-clean PageRank-specialized implementation after enabling
  nonnegative coefficient/bias simplification and output-register fanout
  replication; the routed WNS is `+0.001 ns`.
- P4 degree8/physical8 is the current fair conventional FPGA arithmetic
  baseline: one stage, 32 rows in parallel, 8 full-word `32x32` MAC slots per
  row, routed WNS `+1.284 ns`, `1024` DSP, and `17` cycles for 4 feedback
  iterations in the standalone operator checkpoint.
- Current P2/P3 should not be claimed as a latency win over P4.  The defensible
  hardware result today is `0 DSP` and lower routed dynamic power for P2 degree8
  (`0.710 W` vs P4 `3.046 W`) at the cost of higher LUT and longer cycle count.
- P3 dense32 has a valid RTL cycle-model result and synth-only resource trend;
  it must not be presented as a routed timing result until routed reports exist.
- P4 is the conventional FPGA arithmetic baseline.  CPU NumPy/OpenBLAS is a
  software reference and must not be mixed with FPGA utilization rows.
